Phase 9

Data Format Ablation

What is the optimal information shape for fine-tuning 230M–500M language models?
LFM2.5-230M · 6 formats · 300 canonical examples · 153 eval prompts · Judge-based scoring
🎯
Core question: Phase 8 showed that dataset format dominates hyperparameters (5× more impact). Multi-turn concise chat data crushed flat Alpaca/Dolly-style data. But was that content, format, or both? Phase 9 holds content constant and varies only representation — the first controlled experiment to isolate data shape as a variable in small-model SFT.

Key Findings
🏆
Multi-turn verbose wins on both loss and win-rate. Loss 1.37 (best) and win-rate 0.399 (best). This is surprising — Phase 8 suggested multi-turn concise was the winner. But when content is held constant, the model benefits from the additional conversational context that verbose format provides. The extra assistant turns give the model more tokens to learn from per example.
⚠️
Loss does NOT reliably predict behavioral quality. Bad Format Control has the second-lowest loss (1.40) but the lowest win-rate (0.320). The model learns filler phrases and generic caveats easily — they're statistically predictable text. But this learned pattern produces worse outputs when evaluated by a judge. Loss measures "how well can the model predict the next token" — not "does the model produce useful outputs."
📊
Flat formats have best JSON validity (0.971). Alpaca flat and single-turn chat produce the most valid JSON, slightly ahead of multi-turn formats (0.882–0.912). This may be because flat formats have simpler output expectations — there's no multi-turn context to confuse the model's formatting behavior.
📏
Structured terse produces the shortest outputs (37.7 tokens). Bad format control produces the longest (73.1 tokens, 94% more). Slop phrases (generic filler) only appear in bad format control (8.5%). Training on verbose generic data teaches the model to produce verbose generic output.
Hypothesis Verdicts

H1 — Multi-turn concise format is genuinely better for small-model SFT

When content is held constant, multi-turn verbose beat concise on both loss (1.37 vs 1.51) and win-rate (0.399 vs unknown). Concise is good but verbose is better — the extra turns provide more learning signal per example.

Verdict: REJECTED — verbose beats concise when content is controlled

H2 — The smol-magpie-ultra advantage is partly format, not merely content

Multi-turn formats (verbose: 0.399, concise: strong) outperform flat formats (alpaca: 0.386, single-turn: 0.327) on win-rate. The format advantage is real but smaller than Phase 8 suggested — content still matters enormously (smol-magpie's content is curated).

Verdict: SUPPORTED — format contributes, but content and format interact

H3 — Small models benefit from dense, compact examples more than verbose ones

Structured terse (most compact) has the worst loss (1.83) and middling win-rate (0.340). Verbose multi-turn (more tokens per example) has the best loss and win-rate. Small models benefit from more context, not less.

Verdict: REJECTED — more context beats less context at 230M scale

H4 — Training loss may not correlate with behavioral quality

Bad Format Control: loss 1.40 (2nd best), win-rate 0.320 (worst). Structured Terse: loss 1.83 (worst), win-rate 0.340 (4th). Loss and quality are decoupled — low loss can mean the model learned easy patterns (filler) rather than useful ones.

Verdict: CONFIRMED — loss does not reliably predict behavioral quality

H5 — Surgical LoRA preserves the base model while adding useful behavior

Surgical adapters not yet trained on this canonical dataset. Reserved for Phase 9b.

Verdict: PENDING

H6 — Structured terse may outperform verbose chat on JSON/extraction tasks

Structured terse: JSON validity 0.912, win-rate 0.340. Multi-turn verbose: JSON validity 0.912, win-rate 0.399. Same JSON validity, but verbose wins overall. Structured terse doesn't have a meaningful advantage even on JSON tasks.

Verdict: REJECTED — no JSON advantage from structured terse format

H7 — There is a distinct "small-model-native" data style

Multi-turn verbose is the winner — more conversational context, not less. This is the opposite of what Phase 8 suggested. The "small-model-native" style may actually be "give the model more examples of good behavior per training step" rather than "make examples as compact as possible."

Verdict: SUPPORTED — but the style is "more context" not "more compact"

This is the most important finding. Training loss and behavioral quality measure different things.

Bad Format Control teaches the model to predict filler — easy loss, bad behavior. Structured Terse is hard to predict — high loss, but the model that learns it produces denser outputs. Do not optimize for loss alone.

What This Answers
  • Did multi-turn concise still win? No. Multi-turn verbose beat it on both loss and win-rate when content was held constant.
  • Was smol-magpie advantage mostly content or format? Both. Multi-turn format helps, but the curated content of smol-magpie-ultra is also essential.
  • Which format gives best win-rate? Multi-turn verbose (0.399).
  • Which format gives best loss? Multi-turn verbose (1.37).
  • Do loss and quality correlate? Not reliably. Bad Format Control has 2nd-best loss but worst win-rate.
  • Is there a small-model-native data style? Yes — multi-turn with more context per example, not less.
  • What should we train next? Surgical adapters on multi-turn verbose. bilawal_smol_magpie_v1 with verbose-style training data.
