The Production Recipe

📊 SFT: 200 examples, multi-turn concise

smol-magpie-ultra format. Adafactor, lr=2e-4, LoRA r=8 on hub layers, 300 steps. Gets you 67% clean accuracy reliably. Going to 500 examples costs 2.5× more data curation for 80%.

🧪 Add 100 messy extraction pairs

Mix structured extraction examples with informal/messy inputs. Doubles messy accuracy (20% → 40%) without hurting clean performance. Only add if your deployment sees messy text.

🚀 Deploy with noise + K=5 for critical outputs

Embedding noise σ=0.01 + best-of-5 selection takes 67% → 100% on clean prompts at 5× compute. Use K=2 for cost-sensitive pipelines (still +13pp). σ=0.005 for conservative production.

❌ Don't bother with

Curriculum ordering (irrelevant at 200 examples). Layer-specific noise (destructive on conv architectures). Q-head training (model already too good). More than 500 clean examples.

Decision Tree

Model baseline < 50%? → Noise + K=5 is transformative. Do it first, then SFT.

Model baseline 50-80%? → SFT will help more than noise. Do SFT first, then noise as polish.

Model baseline > 80%? → Noise adds 3-7pp. Only worth it for messy inputs or critical outputs.

Deploying on messy real-world text? → Always add 100 messy pairs to SFT data + noise + K=5.

Methodology

Model: LiquidAI/LFM2.5-230M (instruction-tuned, 230M params, 14 hybrid conv+attn layers)

Training: LoRA r=8, lora_alpha=16, hub layers (L0, L2, L4, L5), Adafactor lr=2e-4, 300 steps, batch=2, grad_accum=4

Training data: smol-magpie-ultra (HuggingFaceTB/smoltalk) multi-turn concise format

Eval prompts: 15 clean structured extraction + 15 messy/informal extraction prompts (held out)

Decoding: temperature=0.2, top_p=0.9, max_new_tokens=64

Noise: Gaussian embedding noise at σ=0.01, K=5 rollouts, best-of-K by field recall

Hardware: aero (RTX 2070 Super 8GB)

Total compute: ~3.3 hours across all experiments
